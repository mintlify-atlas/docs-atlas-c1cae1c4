## Overview

After building your HNG React application, you can deploy it to various hosting platforms. This guide covers the most popular options with detailed deployment steps.

<Note>
Before deploying, ensure you've created a production build by running `npm run build`. See the [Building for Production](/deployment/building) guide for details.
</Note>

## Deployment Platforms

### Vercel (Recommended)

Vercel offers the fastest and easiest deployment experience for React applications with automatic builds and deployments.

<Steps>
  <Step title="Install Vercel CLI">
    ```bash
    npm install -g vercel
    ```
  </Step>
  
  <Step title="Navigate to your project">
    ```bash
    cd /path/to/hng-project
    ```
  </Step>
  
  <Step title="Deploy">
    ```bash
    vercel
    ```
    Follow the prompts to link your project and configure settings.
  </Step>
  
  <Step title="Deploy to production">
    ```bash
    vercel --prod
    ```
  </Step>
</Steps>

#### Automatic Deployments with Git

<Steps>
  <Step title="Push to GitHub">
    Push your code to a GitHub repository
  </Step>
  
  <Step title="Import to Vercel">
    Visit [vercel.com](https://vercel.com) and click "New Project"
  </Step>
  
  <Step title="Configure build settings">
    Vercel automatically detects Create React App:
    - **Build Command**: `npm run build`
    - **Output Directory**: `build`
    - **Install Command**: `npm install`
  </Step>
  
  <Step title="Add environment variables">
    Configure production environment variables in the Vercel dashboard under Settings → Environment Variables
  </Step>
  
  <Step title="Deploy">
    Click "Deploy" - future pushes to your main branch will auto-deploy
  </Step>
</Steps>

### Netlify

Netlify provides excellent static site hosting with continuous deployment and built-in CI/CD.

#### Deploy via Netlify CLI

<Steps>
  <Step title="Install Netlify CLI">
    ```bash
    npm install -g netlify-cli
    ```
  </Step>
  
  <Step title="Build your project">
    ```bash
    npm run build
    ```
  </Step>
  
  <Step title="Deploy">
    ```bash
    netlify deploy
    ```
    Select "build" as your deploy folder
  </Step>
  
  <Step title="Deploy to production">
    ```bash
    netlify deploy --prod
    ```
  </Step>
</Steps>

#### Deploy via Git Integration

<Steps>
  <Step title="Connect repository">
    Go to [netlify.com](https://netlify.com) and click "New site from Git"
  </Step>
  
  <Step title="Configure build settings">
    - **Build command**: `npm run build`
    - **Publish directory**: `build`
  </Step>
  
  <Step title="Add environment variables">
    In Site settings → Build & deploy → Environment, add your `REACT_APP_*` variables
  </Step>
  
  <Step title="Deploy">
    Click "Deploy site" - automatic deployments enabled for all commits
  </Step>
</Steps>

#### Netlify Redirects Configuration

For single-page applications with client-side routing, create `public/_redirects`:

```plaintext
/*    /index.html   200
```

This ensures all routes are handled by React Router.

### GitHub Pages

Deploy directly to GitHub Pages for free static hosting.

<Steps>
  <Step title="Install gh-pages package">
    ```bash
    npm install --save-dev gh-pages
    ```
  </Step>
  
  <Step title="Add homepage to package.json">
    ```json
    {
      "homepage": "https://yourusername.github.io/hng"
    }
    ```
  </Step>
  
  <Step title="Add deployment scripts">
    Add these scripts to `package.json`:
    ```json
    "scripts": {
      "predeploy": "npm run build",
      "deploy": "gh-pages -d build"
    }
    ```
  </Step>
  
  <Step title="Deploy">
    ```bash
    npm run deploy
    ```
    This builds and pushes to the `gh-pages` branch
  </Step>
  
  <Step title="Configure GitHub Pages">
    In your repository settings, set GitHub Pages source to the `gh-pages` branch
  </Step>
</Steps>

<Warning>
GitHub Pages doesn't natively support client-side routing. You may need to use hash routing or implement a 404.html workaround for proper route handling.
</Warning>

### AWS S3 + CloudFront

For enterprise-grade hosting with AWS infrastructure.

<Steps>
  <Step title="Build your application">
    ```bash
    npm run build
    ```
  </Step>
  
  <Step title="Create S3 bucket">
    - Go to AWS S3 Console
    - Create a new bucket with a unique name
    - Enable static website hosting
    - Set index.html as the index document
  </Step>
  
  <Step title="Configure bucket policy">
    Add public read access:
    ```json
    {
      "Version": "2012-10-17",
      "Statement": [
        {
          "Sid": "PublicReadGetObject",
          "Effect": "Allow",
          "Principal": "*",
          "Action": "s3:GetObject",
          "Resource": "arn:aws:s3:::your-bucket-name/*"
        }
      ]
    }
    ```
  </Step>
  
  <Step title="Upload build files">
    Using AWS CLI:
    ```bash
    aws s3 sync build/ s3://your-bucket-name --delete
    ```
  </Step>
  
  <Step title="Create CloudFront distribution (Optional)">
    - Set S3 bucket as origin
    - Configure custom error responses (404 → /index.html)
    - Enable HTTPS with ACM certificate
  </Step>
</Steps>

### DigitalOcean App Platform

<Steps>
  <Step title="Connect GitHub repository">
    Go to DigitalOcean App Platform and create a new app

  <Step title="Configure build settings">
    - **Build Command**: `npm run build`

## Environment Variables Configuration

Properly managing environment variables is crucial for production deployments.

### Creating Environment Files

Create separate environment files for different stages:

```bash
.env.local          # Local development (gitignored)
.env.development    # Development builds
.env.production     # Production builds
```

### Example Production Variables

```bash
# .env.production
REACT_APP_API_URL=https://api.production.com
REACT_APP_ENV=production
REACT_APP_ENABLE_ANALYTICS=true
REACT_APP_SENTRY_DSN=https://your-sentry-dsn
```

### Platform-Specific Configuration

**Vercel:**
```bash
vercel env add REACT_APP_API_URL production
```

**Netlify:**
Add in UI: Site settings → Build & deploy → Environment

**GitHub Actions:**
```yaml
env:
  REACT_APP_API_URL: ${{ secrets.REACT_APP_API_URL }}
```

<Warning>
Never commit `.env.local` or `.env.production` files with sensitive data to version control. Use platform-specific secret management instead.
</Warning>

## Production Best Practices

### 1. Enable HTTPS

Always use HTTPS in production:
- Vercel and Netlify provide automatic HTTPS
- For custom domains, use Let's Encrypt or platform SSL certificates
- Redirect all HTTP traffic to HTTPS

### 2. Configure Caching Headers

Optimize caching for static assets:

**Netlify** (`netlify.toml`):
```toml
[[headers]]
  for = "/static/*"
  [headers.values]
    Cache-Control = "public, max-age=31536000, immutable"

[[headers]]
  for = "/index.html"
  [headers.values]
    Cache-Control = "public, max-age=0, must-revalidate"
```

**Vercel** (`vercel.json`):
```json
{
  "headers": [
    {
      "source": "/static/(.*)",
      "headers": [
        {
          "key": "Cache-Control",
          "value": "public, max-age=31536000, immutable"
        }
      ]
    }
  ]
}
```

### 3. Implement Error Tracking

Integrate error monitoring for production:

```javascript
// src/index.js
import * as Sentry from "@sentry/react";

if (process.env.NODE_ENV === 'production') {
  Sentry.init({
    dsn: process.env.REACT_APP_SENTRY_DSN,
    environment: process.env.REACT_APP_ENV,
  });
}
```

### 4. Enable Compression

Most platforms enable gzip/brotli automatically. Verify compression:

```bash
curl -H "Accept-Encoding: gzip" -I https://your-domain.com
```

Look for `Content-Encoding: gzip` or `Content-Encoding: br` in headers.

### 5. Set Up Performance Monitoring

Monitor Web Vitals in production:

```javascript
// src/index.js
import { getCLS, getFID, getFCP, getLCP, getTTFB } from 'web-vitals';

function sendToAnalytics(metric) {
  // Send to your analytics endpoint
  console.log(metric);
}

getCLS(sendToAnalytics);
getFID(sendToAnalytics);
getFCP(sendToAnalytics);
getLCP(sendToAnalytics);
getTTFB(sendToAnalytics);
```

### 6. Configure Custom Domain

**Vercel:**
<Steps>
  <Step title="Add domain in dashboard">
    Go to Project Settings → Domains
  </Step>
  <Step title="Add DNS records">
    Add A record or CNAME as specified by Vercel
  </Step>
  <Step title="Verify domain">
    Vercel automatically provisions SSL certificate
  </Step>
</Steps>

**Netlify:**
<Steps>
  <Step title="Add custom domain">
    Site settings → Domain management → Add custom domain
  </Step>
  <Step title="Update DNS">
    Point DNS to Netlify's load balancer
  </Step>
  <Step title="Enable HTTPS">
    Netlify automatically provisions Let's Encrypt certificate
  </Step>
</Steps>

## CI/CD Pipeline

### GitHub Actions Example

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy to Production

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run tests
        run: npm test -- --coverage --watchAll=false
      
      - name: Build
        run: npm run build
        env:
          REACT_APP_API_URL: ${{ secrets.REACT_APP_API_URL }}
      
      - name: Deploy to Netlify
        uses: netlify/actions/cli@master
        env:
          NETLIFY_AUTH_TOKEN: ${{ secrets.NETLIFY_AUTH_TOKEN }}
          NETLIFY_SITE_ID: ${{ secrets.NETLIFY_SITE_ID }}
        with:
          args: deploy --prod --dir=build
```

## Deployment Checklist

Before deploying to production:

- [ ] Run `npm run build` locally and test with `serve -s build`
- [ ] All tests passing (`npm test`)
- [ ] Environment variables configured correctly
- [ ] Remove console.log statements and debug code
- [ ] Update version number in package.json
- [ ] Check bundle size with `npm run analyze`
- [ ] Verify all images are optimized
- [ ] Test on multiple browsers and devices
- [ ] Configure error tracking (Sentry, etc.)
- [ ] Set up analytics (Google Analytics, etc.)
- [ ] Configure custom domain and HTTPS
- [ ] Set up proper caching headers
- [ ] Create backup/rollback strategy
- [ ] Document deployment process for team

## Rollback Strategy

Most platforms support instant rollbacks:

**Vercel:**
```bash
vercel rollback [deployment-url]
```

**Netlify:**
UI: Deploys → Click on previous deploy → Publish deploy

**GitHub Pages:**
```bash
git revert HEAD
npm run deploy
```

## Monitoring and Maintenance

After deployment:

1. **Monitor Error Rates**: Set up alerts for increased error rates
2. **Track Performance**: Monitor Core Web Vitals and loading times
3. **Check Uptime**: Use services like UptimeRobot or Pingdom
4. **Review Logs**: Regularly check deployment and runtime logs
5. **Update Dependencies**: Keep packages up-to-date for security

## Troubleshooting

### Blank Page After Deployment

- Check browser console for errors
- Verify `homepage` field in package.json matches deployment URL
- Ensure all assets are being served correctly
- Check for routing issues with SPA configuration

### Environment Variables Not Working

- Verify variables start with `REACT_APP_`
- Rebuild after changing environment variables
- Check platform-specific variable configuration
- Clear build cache and rebuild

### 404 Errors on Page Refresh

- Configure SPA fallback to index.html
- Add `_redirects` file for Netlify
- Set up CloudFront custom error responses for AWS
- Use hash router as fallback for platforms without SPA support
